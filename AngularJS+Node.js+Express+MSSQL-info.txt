=========================# Product CRUD — AngularJS + Node.js + Express + Microsoft SQL Server======================

## Run it
1. Start Microsoft SQL Server and ensure TCP/IP connections are enabled (typically port `1433`).
2. Copy `.env.example` to `.env`, then set your SQL Server host/instance and login details. For a SQL Server Express instance, set `DB_SERVER=localhost`, `DB_INSTANCE=SQLEXPRESS`, and leave `DB_PORT` blank if it uses dynamic ports.
3. Run:

```powershell
npm.cmd install
npm.cmd start
```

Open http://localhost:3000.

The REST API is available at `/api/products` and supports `GET`, `POST`, `PUT /:id`, and `DELETE /:id`. Product records are persisted in the SQL Server `product_crud.dbo.products` table. The server creates the database and table automatically.

The default connection uses `localhost:1433` and SQL Server authentication. Set `DB_SERVER`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, and (if needed) `DB_ENCRYPT` in `.env`. Your login needs permission to create a database; if it does not, create `product_crud` manually and grant the login access to it.

The standalone SQL Server schema is in `database/schema.sql`. Open it in SQL Server Management Studio and execute it to create the database and table manually.





==============How to connect MSSQL Server/SQL Express to your project===================

1. Connect as an admin first
	Of your two available connections, expand localhost\SQLEXPRESS (... - DESKTOP-0EBOC35\BDC computer) — Double-click it to connect.
connect.

2. Open a New Query on that connection, and run this to create a fresh, trouble-free login:

sql
	-- Create the login with policy checks turned OFF (avoids the expiration issue)
	CREATE LOGIN AppUser WITH PASSWORD = 'YourStrongP@ss123', 
    		CHECK_POLICY = OFF, 
    		CHECK_EXPIRATION = OFF;
	GO
	
	-- Switch to your app's database
	USE product_crud;
	GO
	
	-- Create a user mapped to that login
	CREATE USER AppUser FOR LOGIN AppUser;
	GO
	
	-- Grant it read/write permissions (adjust as needed)
	ALTER ROLE db_datareader ADD MEMBER AppUser;
	ALTER ROLE db_datawriter ADD MEMBER AppUser;
	GO

3. Enable SQL Authentication on the server
	Right-click the server root (BDC computer) → Properties → Security → select "SQL Server and Windows Authentication mode"

4. Update your .env:

	DB_SERVER=localhost
	DB_INSTANCE=SQLEXPRESS
	DB_PORT=1433
	DB_USER=AppUser
	DB_PASSWORD=YourStrongP@ss123
	DB_NAME=product_crud
	DB_ENCRYPT=false
	PORT=3000

5. npm start
